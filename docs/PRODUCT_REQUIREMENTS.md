# 2. 产品定位

## 2.1 战略定位

星镜剧创定位为：

> 面向中国 AI 短剧 / 漫剧团队的生产操作系统。

平台不是单点 AI 生图、生视频工具，也不是简单剪辑器，而是围绕短剧 / 漫剧生产链路，提供从小说 / 剧本解析、AI 导演理解、角色场景资产、智能分镜、故事板、多模型生成、配音对口型、审片协作、合规留痕，到成片导出和多平台发布包交付的一体化生产能力。

## 2.2 产品一句话定义

> 星镜剧创是一款 AI 短剧 / 漫剧工业化生产工作台，帮助个人创作者、AI 短剧工作室、MCN、品牌方、IP 方和影视内容团队完成从小说 / 剧本到可审核、可投放、可复用、可交付成片的全流程生产。

## 2.3 产品不是什么

星镜剧创不是：

1. 不是单点 AI 生图 / 生视频站。
2. 不是简单的短视频剪辑器。
3. 不是播放分发平台。
4. 不是只面向 C 端玩票用户的娱乐工具。
5. 不是完全替代剪映、Premiere、DaVinci 的专业 NLE。
6. 不是承诺“一键全自动生成商业精品剧”的黑盒工具。
7. 不是鼓励侵权改编、搬运、未授权 IP 二创的平台。

## 2.4 核心价值

| 核心价值 | 说明 |
|---|---|
| 生产流程统一 | 把剧本、分镜、角色资产、生成任务、配音字幕、审片、导出放到同一项目中 |
| 角色资产一致 | 通过主体资产库、参考图、多状态、多版本、引用关系解决多镜头崩坏 |
| 多模型可控 | 不绑定单一模型，通过模型中心和路由策略控制质量、成本、速度 |
| 团队协作 | 支持工作室分工、审片评论、任务状态、版本记录、客户审阅 |
| 成本透明 | 生成前预估，生成中追踪，成功后扣费，失败有记录和退回策略 |
| 合规留痕 | 记录模型、提示词、素材来源、AIGC 标识、授权状态、导出记录 |
| 发布交付 | 输出 MP4、字幕、分镜表、素材包、封面、标题、平台发布包 |
| 可复用沉淀 | 项目模板、分镜模板、角色模板、提示词模板、风格模板、Fork 二创 |
| 平台可运营 | 后台可管理用户、模型、算力、订单、审核、合规、模板、商单、客服和数据 |

# 8. 前台详细功能需求

## 8.1 账号认证模块

功能目标：让用户安全登录平台，并支持个人空间和团队空间切换。

功能清单：

| 编号 | 功能 | 优先级 |
|---|---|---|
| ACC-001 | 手机号验证码登录 | P0 |
| ACC-002 | 邮箱密码登录 | P0 |
| ACC-003 | 邮箱验证码登录 | P0 |
| ACC-004 | 微信登录 | P1 |
| ACC-005 | GitHub / Google / Apple 登录 | P2 |
| ACC-006 | 忘记密码 | P0 |
| ACC-007 | 修改密码 | P0 |
| ACC-008 | 登录设备管理 | P1 |
| ACC-009 | 账号注销 | P1 |
| ACC-010 | 风险登录提醒 | P1 |

验收标准：

1. 用户可以在 30 秒内完成注册登录。
2. 用户登录后可进入默认工作区。
3. Token 过期后可刷新。
4. 被禁用用户不能登录。
5. 登录日志可追踪。

功能目标：支持个人创作者和团队型工作室共同使用。

功能清单：

| 编号 | 功能 | 优先级 |
|---|---|---|
| WS-001 | 创建个人空间 | P0 |
| WS-002 | 创建团队空间 | P0 |
| WS-003 | 邀请成员 | P0 |
| WS-004 | 成员角色设置 | P0 |
| WS-005 | 移除成员 | P0 |
| WS-006 | 席位管理 | P1 |
| WS-007 | 团队套餐绑定 | P1 |
| WS-008 | 团队账单归属 | P1 |
| WS-009 | 团队操作日志 | P1 |

验收标准：

1. 团队所有者可以邀请成员加入。

5. 团队空间可与个人空间清晰切换。

# 12. 异常流程

| 异常 | 处理 |
|---|---|
| 剧本无法解析 | 提示原因，允许手动分集 |
| 角色提取错误 | 用户可合并 / 删除 / 新增 |
| 主体图不满意 | 重抽、换模型、上传参考图 |
| 分镜过碎 | 合并建议 |
| 分镜过长 | 拆分建议 |
| 图片生成失败 | 重试 / 换模型 |
| 视频生成失败 | 重试 / 降级模型 |
| 算力不足 | 提示充值 / 降低质量 |
| 任务排队过久 | 展示队列 / 切换模型 |
| 对口型失败 | 保留原视频 |
| 字幕不同步 | 重新对齐 |
| 导出失败 | 后台重试 |
| 合规不通过 | 阻断正式导出 |
| 文件丢失 | 重新生成或提示恢复 |
| 权限不足 | 返回明确错误 |
| 客户误操作 | 外链只读控制 |
| 后台误操作 | 审计记录，可按策略回滚 |
| 异常扣费 | 工单 + 后台退费流程 |
| 模型大面积失败 | 模型下线 + 路由切换 + 公告 |

# 15. 验收总标准

## 15.1 产品主流程验收

1. 用户可以创建项目。
2. 用户可以导入剧本。
3. 系统可以完成内容体检。
4. 系统可以提取角色、场景、道具。
5. 用户可以生成角色 / 场景图。
6. 系统可以生成分镜。
7. 用户可以编辑分镜。
8. 系统可以生成故事板。
9. 用户可以批量生成图片。
10. 用户可以批量生成视频。
11. 用户可以生成配音和字幕。
12. 用户可以自动成片。
13. 用户可以导出 MP4。
14. 用户可以查看算力消耗。
15. 生成失败可以重试。
16. 导出前可以进行合规检查。

## 15.2 团队协作验收

1. 团队管理员可以邀请成员。
2. 成员可以按权限访问项目。
3. 审核者可以评论和打回。
4. 客户外链只能审片。
5. 操作日志可查。

## 15.3 商业化验收

1. 生成前可预估算力。
2. 任务提交后可冻结算力。
3. 成功后扣费。
4. 失败后退回。
5. 项目成本可统计。
6. 用户可查看账单流水。
7. 后台可处理异常扣费和退款。

## 15.4 合规验收

1. 模型调用有记录。
2. 提示词有记录。
3. 生成结果有记录。
4. 素材来源可标记。
5. AIGC 标识策略可记录。
6. 导出操作有审计日志。
7. 合规报告可导出。
8. 后台可阻断高风险项目导出。

## 15.5 平台后台验收

1. 后台员工可以独立登录。
2. 后台角色权限可配置。
3. 可管理用户、团队、项目、任务、模型、算力、订单、审核、合规、模板、商单、客服。
4. 所有敏感操作有日志。
5. 高危操作需要二次确认。
6. 后台数据可按权限导出。
7. 平台运营可以通过后台完成日常运营管理。
